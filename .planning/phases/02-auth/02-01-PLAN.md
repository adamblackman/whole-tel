<task type="auto">
  <name>Task 2: Create src/proxy.ts for session refresh</name>
  <files>src/proxy.ts</files>
  <action>
Create `src/proxy.ts` at the root of the `src/` directory (same level as `src/app/`). This is Next.js 16's replacement for `middleware.ts` — the function must be named `proxy` (not `middleware`).

The proxy must:
1. Create a Supabase client with BOTH request cookie reads AND response cookie writes
2. Call `supabase.auth.getUser()` to trigger token refresh (NOT getSession())
3. Return the response with updated cookies set

Critical: The `setAll` handler must write to request.cookies first (so Server Components see refreshed token), then rebuild the response with the updated request, then write to response.cookies (so the browser stores the refreshed token). This two-write pattern is what makes session refresh work end-to-end.

```typescript
// src/proxy.ts
import { createServerClient, type CookieOptions } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function proxy(request: NextRequest) {
  let response = NextResponse.next({
    request: {
      headers: request.headers,
    },
  })

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return request.cookies.getAll()
        },
        setAll(cookiesToSet: { name: string; value: string; options: CookieOptions }[]) {
          // Step 1: Write to request cookies so Server Components see refreshed token
          cookiesToSet.forEach(({ name, value }) =>
            request.cookies.set(name, value)
          )
          // Step 2: Rebuild response with updated request headers
          response = NextResponse.next({
            request,
          })
          // Step 3: Write to response cookies so browser stores refreshed token
          cookiesToSet.forEach(({ name, value, options }) =>
            response.cookies.set(name, value, options)
          )
        },
      },
    }
  )

  // Trigger token refresh — use getUser() NOT getSession()
  await supabase.auth.getUser()

  return response
}

export const config = {
  matcher: [
    '/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)',
  ],
}
```

Do NOT add role-based redirect logic here. Proxy = session refresh only. Route protection belongs in the DAL via layout.tsx files.

<task type="auto">
  <name>Task 3: Create Server Actions for all auth mutations</name>
  <files>src/lib/actions/auth.ts</files>
  <action>
Create `src/lib/actions/auth.ts`. This file uses `'use server'` directive and must ONLY import from `@/lib/supabase/server` (not browser.ts — Server Actions run on the server, localStorage doesn't exist there).

Implement these five exports:

**signUpGuest(formData: FormData)**
- Reads: email, password, display_name from formData
- Calls: supabase.auth.signUp({ email, password, options: { data: { role: 'guest', display_name } } })
- On error: return { error: error.message }
- On success: revalidatePath('/', 'layout'), redirect('/')

**signUpOwner(formData: FormData)**
- Reads: email, password from formData
- Calls: supabase.auth.signUp({ email, password, options: { data: { role: 'owner' } } })
- Owner signup at /owner/signup is open registration (no secret URL — v1 decision)
- On error: return { error: error.message }
- On success: redirect('/owner/login?message=Account+created.+Sign+in+to+access+your+dashboard.')

**signIn(formData: FormData)**
- Reads: email, password, return_to (optional) from formData
- Calls: supabase.auth.signInWithPassword({ email, password })
- On error: return { error: error.message }
- On success: revalidatePath('/', 'layout'), redirect(returnTo || '/')

**signInAsOwner(formData: FormData)**
- Reads: email, password from formData
- Calls: supabase.auth.signInWithPassword({ email, password })
- On auth error: return { error: error.message }
- On auth success: query profiles table WHERE id = user.id, select role
- If profile.role !== 'owner': call supabase.auth.signOut(), return { error: 'This account does not have owner access.' }
- On owner verified: revalidatePath('/', 'layout'), redirect('/dashboard')

**signOut()**
- Calls: supabase.auth.signOut()
- Then: revalidatePath('/', 'layout'), redirect('/')
- revalidatePath MUST be called before redirect — otherwise the page cache retains the logged-in state after logout

Return type pattern: `Promise<{ error: string } | void>` — return { error } on failure, void (implicit) on success + redirect.
