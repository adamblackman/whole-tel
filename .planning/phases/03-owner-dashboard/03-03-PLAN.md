Wait — Server Components cannot call `.bind()` on Server Actions directly in JSX. The PropertyForm Client Component must handle the binding internally.

Updated approach: PropertyForm accepts an optional `propertyId` prop. When `propertyId` is provided, it imports `updateProperty` and binds internally:

```tsx
// In PropertyForm.tsx (Client Component):
import { createProperty, updateProperty } from '@/lib/actions/properties'

// If propertyId is provided, use updateProperty.bind(null, propertyId)
// Otherwise use createProperty
const action = propertyId
  ? updateProperty.bind(null, propertyId)
  : createProperty
const [state, formAction, pending] = useActionState(action, {})
```

Actually, simpler: the parent page passes the action as a prop. But Server Actions can be passed as props from Server Components to Client Components. So:

```tsx
// edit/page.tsx (Server Component):
import { updateProperty } from '@/lib/actions/properties'

const updateWithId = updateProperty.bind(null, propertyId)
return <PropertyForm action={updateWithId} initialData={property} submitLabel="Save Changes" />
```

This works because `.bind()` on a Server Action in a Server Component produces a new Server Action reference that can be passed to a Client Component. This is the documented Next.js pattern.

- `<PropertyForm action={updateWithId} initialData={{ ...property, amenities: property.amenities as string[] }} submitLabel="Save Changes" />`

Parse the property data into the initialData shape expected by PropertyForm:
