Create `src/lib/actions/photos.ts` with `'use server'` directive.

```typescript
'use server'

import { revalidatePath } from 'next/cache'
import { createClient } from '@/lib/supabase/server'
import { requireOwner } from '@/lib/dal'
```

Implement three exports:

**getSignedUploadUrl(propertyId: string): Promise<{ signedUrl: string; token: string; path: string } | { error: string }>**
1. `const user = await requireOwner()`
2. `const supabase = await createClient()`
3. Verify the property belongs to this owner:
   ```typescript
   const { data: prop } = await supabase
     .from('properties')
     .select('id')
     .eq('id', propertyId)
     .eq('owner_id', user.id)
     .single()
   if (!prop) return { error: 'Property not found or access denied' }
   ```
4. Generate a unique storage path using `crypto.randomUUID()`:
   ```typescript
   const path = `${user.id}/${propertyId}/${crypto.randomUUID()}`
   ```
   Note: No file extension in the path — Supabase Storage handles content-type from the uploaded file headers. Using `crypto.randomUUID()` prevents the collision pitfall mentioned in research.
5. Create the signed upload URL:
   ```typescript
   const { data, error } = await supabase.storage
     .from('property-photos')
     .createSignedUploadUrl(path)
   if (error) return { error: error.message }
   return { signedUrl: data.signedUrl, token: data.token, path }
   ```

**savePhotoRecord(propertyId: string, storagePath: string): Promise<{ error?: string }>**
1. `await requireOwner()` — the RLS policy on property_photos.INSERT will verify ownership
2. `const supabase = await createClient()`
3. Count existing photos to determine display_order:
   ```typescript
   const { count } = await supabase
     .from('property_photos')
     .select('*', { count: 'exact', head: true })
     .eq('property_id', propertyId)
   ```
4. Insert the photo record:
   ```typescript
   const { error } = await supabase
     .from('property_photos')
     .insert({
       property_id: propertyId,
       storage_path: storagePath,
       display_order: (count ?? 0),
     })
   if (error) return { error: error.message }
   ```
5. Revalidate:
   ```typescript
   revalidatePath(`/dashboard/properties/${propertyId}`)
   revalidatePath('/dashboard')
   return {}
   ```

**deletePhoto(photoId: string, propertyId: string, storagePath: string): Promise<{ error?: string }>**
1. `await requireOwner()`
2. `const supabase = await createClient()`
3. Delete from storage first:
   ```typescript
   const { error: storageError } = await supabase.storage
     .from('property-photos')
     .remove([storagePath])
   // Log but don't block on storage deletion failure — the DB record removal is more important
   ```
4. Delete the database record:
   ```typescript
   const { error: dbError } = await supabase
     .from('property_photos')
     .delete()
     .eq('id', photoId)
   if (dbError) return { error: dbError.message }
   ```
   RLS DELETE policy enforces ownership via EXISTS subquery.
5. Revalidate:
   ```typescript
   revalidatePath(`/dashboard/properties/${propertyId}`)
   revalidatePath('/dashboard')
   return {}
   ```

**Implementation details:**

1. **Imports:**
   - `createClient` from `@/lib/supabase/browser` (for direct Storage upload)
   - `getSignedUploadUrl`, `savePhotoRecord`, `deletePhoto` from `@/lib/actions/photos`
   - `Button` from `@/components/ui/button`
   - `useState`, `useRef` from `react`
   - `Upload`, `Trash2`, `Loader2` from `lucide-react`
   - `Image` from `next/image`

2. **State:**
   - `uploading: boolean` — tracks upload in progress
   - `error: string | null` — displays upload/delete errors
   - `fileInputRef: useRef<HTMLInputElement>` — hidden file input reference

3. **getPublicUrl helper:**
   ```typescript
   function getPublicUrl(storagePath: string): string {
     const supabase = createClient()
     const { data } = supabase.storage
       .from('property-photos')
       .getPublicUrl(storagePath)
     return data.publicUrl
   }
   ```

4. **handleUpload function:**
   ```typescript
   async function handleUpload(e: React.ChangeEvent<HTMLInputElement>) {
     const file = e.target.files?.[0]
     if (!file) return

     setUploading(true)
     setError(null)

     try {
       // Step 1: Get signed URL from server
       const result = await getSignedUploadUrl(propertyId)
       if ('error' in result) throw new Error(result.error)

       // Step 2: Upload directly to Supabase Storage from browser
       const supabase = createClient()
       const { error: uploadError } = await supabase.storage
         .from('property-photos')
         .uploadToSignedUrl(result.path, result.token, file)
       if (uploadError) throw uploadError

       // Step 3: Save the photo record in the database
       const saveResult = await savePhotoRecord(propertyId, result.path)
       if (saveResult.error) throw new Error(saveResult.error)
     } catch (err) {
       setError(err instanceof Error ? err.message : 'Upload failed')
     } finally {
       setUploading(false)
       // Reset file input so the same file can be re-selected
       if (fileInputRef.current) fileInputRef.current.value = ''
     }
   }
   ```

5. **handleDelete function:**
   ```typescript
   async function handleDelete(photoId: string, storagePath: string) {
     setError(null)
     try {
       const result = await deletePhoto(photoId, propertyId, storagePath)
       if (result.error) throw new Error(result.error)
     } catch (err) {
       setError(err instanceof Error ? err.message : 'Delete failed')
     }
   }
   ```

6. **Render:**
   - Section heading: "Photos" with an upload button
   - Hidden `<input type="file" accept="image/*" onChange={handleUpload} ref={fileInputRef} />`
   - Upload button: `<Button variant="outline" size="sm" onClick={() => fileInputRef.current?.click()} disabled={uploading}>` with Upload icon
   - While uploading: show Loader2 spinner icon with "Uploading..." text
   - Error display: `{error && <p className="text-sm text-destructive mt-2">{error}</p>}`
   - Photo grid: `<div className="grid grid-cols-2 md:grid-cols-3 gap-4 mt-4">`
     - Each photo: a relative container with `<Image src={getPublicUrl(photo.storage_path)} alt="Property photo" fill className="object-cover rounded-lg" sizes="(max-width: 768px) 50vw, 33vw" />`
     - Delete button overlay: positioned absolute top-right, small destructive ghost button with Trash2 icon
     - `onClick={() => handleDelete(photo.id, photo.storage_path)}`
   - If no photos: show a muted message "No photos yet. Upload your first photo."

**CRITICAL:** Never route the file through a form action or Server Action body. The `<input type="file">` is NOT inside a `<form>`. The file goes directly from the browser to Supabase Storage.

Use `next/image` with `unoptimized` prop if Supabase Storage URLs are not configured in `next.config.ts` `images.remotePatterns`. Alternatively, check if `next.config.ts` already has the Supabase hostname — if not, add a note in the done criteria that the executor should add the Supabase project hostname to `images.remotePatterns`.
