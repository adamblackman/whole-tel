**src/lib/validations/property.ts**

```typescript
import { z } from 'zod'

export const PropertySchema = z.object({
  name: z.string().min(1, 'Property name is required'),
  description: z.string().optional(),
  location: z.string().min(1, 'Location is required'),
  address: z.string().optional(),
  bedrooms: z.coerce.number().int().min(1, 'At least 1 bedroom required'),
  bathrooms: z.coerce.number().int().min(1, 'At least 1 bathroom required'),
  max_guests: z.coerce.number().int().min(1, 'At least 1 guest required'),
  nightly_rate: z.coerce.number().min(1, 'Nightly rate must be at least $1'),
  cleaning_fee: z.coerce.number().min(0).default(0),
  amenities: z.string().optional(),   // JSON string from form — parsed to array in action
  house_rules: z.string().optional(),
  check_in_time: z.string().default('3:00 PM'),
  check_out_time: z.string().default('11:00 AM'),
})

export type ActionState = {
  errors?: Record<string, string[]>
  message?: string
}
```

Notes:
- `z.coerce.number()` is essential — FormData values are always strings, coerce converts them to numbers for validation.
- `amenities` is accepted as a string (comma-separated or JSON) from the form. The Server Action will parse it into a JSON array before inserting.
- Export `ActionState` from here so all actions share the same return type.

**src/lib/validations/add-on.ts**

```typescript
import { z } from 'zod'

export const AddOnSchema = z.object({
  name: z.string().min(1, 'Add-on name is required'),
  description: z.string().optional(),
  price: z.coerce.number().min(0, 'Price must be $0 or more'),
  pricing_unit: z.enum(['per_person', 'per_booking'], {
    errorMap: () => ({ message: 'Select per person or per booking' }),
  }),
  max_quantity: z.coerce.number().int().positive().optional().or(z.literal('')).transform(val => val === '' ? null : val),
})
```

Notes:
- `pricing_unit` uses `z.enum` matching the database CHECK constraint exactly.
- `max_quantity` accepts empty string (form sends '' for empty optional numeric fields) and transforms to null.

<task type="auto">
  <name>Task 2: Create Server Actions for property CRUD</name>
  <files>src/lib/actions/properties.ts</files>
  <action>
Create `src/lib/actions/properties.ts` with `'use server'` directive.

Import dependencies:
```typescript
'use server'

import { revalidatePath } from 'next/cache'
import { redirect } from 'next/navigation'
import { createClient } from '@/lib/supabase/server'
import { requireOwner } from '@/lib/dal'
import { PropertySchema, type ActionState } from '@/lib/validations/property'
```

Implement three exports:

**createProperty(_prevState: ActionState, formData: FormData): Promise<ActionState>**
1. `const user = await requireOwner()`
2. Parse formData with `PropertySchema.safeParse(Object.fromEntries(formData))`
3. If parse fails: `return { errors: parsed.error.flatten().fieldErrors }`
4. Parse amenities: if `parsed.data.amenities` is a non-empty string, split by commas, trim each, and convert to JSON array. Otherwise default to `[]`.
5. `const supabase = await createClient()`
6. Insert into properties: `{ ...parsed.data, amenities: amenitiesArray, owner_id: user.id }`
   - Remove the raw `amenities` string key before spread to avoid type conflict
7. If insert error: `return { message: error.message }`
8. `revalidatePath('/dashboard')`
9. `redirect('/dashboard')`

CRITICAL: `owner_id` comes from `user.id` — NEVER from formData.

**updateProperty(propertyId: string, _prevState: ActionState, formData: FormData): Promise<ActionState>**
1. `const user = await requireOwner()`
2. Parse formData with PropertySchema (same as create)
3. If parse fails: return errors
4. Parse amenities same as create
5. `const supabase = await createClient()`
6. Update: `.from('properties').update({ ...data, amenities: amenitiesArray }).eq('id', propertyId).eq('owner_id', user.id)`
   - The `.eq('owner_id', user.id)` is UI-layer defense. RLS is the DB-layer defense.
7. If error: return message
8. `revalidatePath('/dashboard')`
9. `revalidatePath(`/dashboard/properties/${propertyId}`)`
10. `redirect(`/dashboard/properties/${propertyId}`)`

Signature note: `propertyId` is the first argument because it will be bound via `updateProperty.bind(null, propertyId)` in the Client Component. `useActionState` then injects `_prevState` as second arg and `formData` as third.

**deleteProperty(propertyId: string): Promise<ActionState>**
1. `const user = await requireOwner()`
2. `const supabase = await createClient()`
3. Delete: `.from('properties').delete().eq('id', propertyId).eq('owner_id', user.id)`
4. If error: `return { message: error.message }`
5. `revalidatePath('/dashboard')`
6. `redirect('/dashboard')`

Note: deleteProperty does NOT use useActionState — it's a direct call from a confirmation dialog button. Signature is simpler (no _prevState, no formData).

Create `src/lib/actions/add-ons.ts` with `'use server'` directive.

Import dependencies:
```typescript
'use server'

import { revalidatePath } from 'next/cache'
import { createClient } from '@/lib/supabase/server'
import { requireOwner } from '@/lib/dal'
import { AddOnSchema } from '@/lib/validations/add-on'
import type { ActionState } from '@/lib/validations/property'
```

Implement three exports:

**createAddOn(propertyId: string, _prevState: ActionState, formData: FormData): Promise<ActionState>**
1. `await requireOwner()` — verifies owner status. RLS will enforce property ownership at DB layer.
2. Parse formData with AddOnSchema.safeParse(Object.fromEntries(formData))
3. If parse fails: return errors
4. `const supabase = await createClient()`
5. Insert into add_ons: `{ ...parsed.data, property_id: propertyId }`
   - RLS policy "Owners can insert add-ons for their properties" uses EXISTS subquery to verify ownership
6. If error: `return { message: error.message }`
7. `revalidatePath(`/dashboard/properties/${propertyId}`)`
8. `return { message: 'Add-on created successfully' }`

Note: No redirect — stay on the property detail page where add-ons are managed inline.

**updateAddOn(addOnId: string, propertyId: string, _prevState: ActionState, formData: FormData): Promise<ActionState>**
1. `await requireOwner()`
2. Parse formData with AddOnSchema
3. If parse fails: return errors
4. `const supabase = await createClient()`
5. Update: `.from('add_ons').update(parsed.data).eq('id', addOnId)`
   - RLS UPDATE policy enforces ownership via EXISTS subquery — no need for manual owner_id check on add_ons table
6. If error: return message
7. `revalidatePath(`/dashboard/properties/${propertyId}`)`
8. `return { message: 'Add-on updated successfully' }`

Signature: addOnId and propertyId are both bound via `.bind(null, addOnId, propertyId)`.

**deleteAddOn(addOnId: string, propertyId: string): Promise<ActionState>**
1. `await requireOwner()`
2. `const supabase = await createClient()`
3. Delete: `.from('add_ons').delete().eq('id', addOnId)`
   - RLS DELETE policy enforces ownership
4. If error: return message
5. `revalidatePath(`/dashboard/properties/${propertyId}`)`
6. `return { message: 'Add-on deleted' }`

Note: deleteAddOn is a direct call (no useActionState) — same pattern as deleteProperty.
