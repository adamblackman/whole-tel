1. Create SQL migration `supabase/migrations/20260324000003_itinerary_schema.sql`:
   - `ALTER TABLE properties ADD COLUMN timezone text NOT NULL DEFAULT 'America/New_York';`
   - `CREATE TABLE property_activities` with columns: id (uuid PK default gen_random_uuid()), property_id (uuid NOT NULL REFERENCES properties(id) ON DELETE CASCADE), name (text NOT NULL), description (text), duration_min (int NOT NULL CHECK > 0), available_slots (jsonb NOT NULL DEFAULT '[]'), is_active (boolean NOT NULL DEFAULT true), created_at (timestamptz NOT NULL DEFAULT now()), updated_at (timestamptz NOT NULL DEFAULT now())
   - `CREATE TABLE itinerary_events` with columns: id (uuid PK default gen_random_uuid()), booking_id (uuid NOT NULL REFERENCES bookings(id) ON DELETE CASCADE), activity_id (uuid REFERENCES property_activities(id) ON DELETE SET NULL), title (text NOT NULL), event_date (date NOT NULL), start_time (time NOT NULL), end_time (time NOT NULL), notes (text), created_at (timestamptz NOT NULL DEFAULT now()), updated_at (timestamptz NOT NULL DEFAULT now()), CONSTRAINT check_event_times CHECK (end_time > start_time)
   - RLS for property_activities: public read (anon + authenticated), owner-scoped ALL using EXISTS subquery on properties.owner_id = auth.uid() (same pattern as property_amenities RLS in Phase 12)
   - RLS for itinerary_events: guest-scoped ALL using EXISTS on bookings.guest_id = auth.uid(), owner SELECT using EXISTS join bookings -> properties.owner_id = auth.uid()
   - Index: `CREATE INDEX idx_itinerary_events_booking ON itinerary_events(booking_id);`
   - Index: `CREATE INDEX idx_property_activities_property ON property_activities(property_id);`

   - Add `TimeSlot` type: { start: string; end: string }
   - Add `ItineraryEvent` interface: id, booking_id, activity_id (string | null), title, event_date, start_time, end_time, notes (string | null), created_at, updated_at

   - TimeSlotSchema: z.object({ start: z.string().regex(/^\d{2}:\d{2}$/), end: z.string().regex(/^\d{2}:\d{2}$/) }).refine(s => s.end > s.start, { message: 'End must be after start' })
