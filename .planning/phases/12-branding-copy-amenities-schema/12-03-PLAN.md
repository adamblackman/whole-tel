Create migration file `supabase/migrations/20260323000001_amenities_schema.sql` with:

**1. Amenities catalog table:**
```sql
CREATE TABLE amenities (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name          text NOT NULL UNIQUE,
  category      text NOT NULL CHECK (category IN ('Water', 'Social', 'Work/Event', 'Culinary', 'Wellness')),
  icon_name     text NOT NULL,
  display_order int NOT NULL DEFAULT 0,
  created_at    timestamptz NOT NULL DEFAULT now()
);
```

**2. Property-amenity join table:**
```sql
CREATE TABLE property_amenities (
  property_id   uuid NOT NULL REFERENCES properties(id) ON DELETE CASCADE,
  amenity_id    uuid NOT NULL REFERENCES amenities(id) ON DELETE CASCADE,
  PRIMARY KEY (property_id, amenity_id)
);
```

**3. RLS policies:**
- `amenities`: Enable RLS. SELECT open to `anon, authenticated` with `USING (true)`.
- `property_amenities`: Enable RLS. SELECT open to `anon, authenticated`. INSERT/UPDATE/DELETE (use FOR ALL) scoped to property owner via `EXISTS (SELECT 1 FROM properties WHERE properties.id = property_id AND properties.owner_id = (SELECT auth.uid()))` for both USING and WITH CHECK.

**4. Tax rate column on properties:**
```sql
ALTER TABLE properties ADD COLUMN tax_rate numeric(5,4);
```
`numeric(5,4)` allows values like 0.1200 (12%), max 9.9999. Nullable -- null means no tax (hotel submits from gross).

**5. Seed amenities data (30 rows across 5 categories):**
Use the exact seed INSERT from the research file -- 30 amenities across Water (7), Social (7), Work/Event (5), Culinary (6), Wellness (6). Use Lucide icon names for `icon_name`.
