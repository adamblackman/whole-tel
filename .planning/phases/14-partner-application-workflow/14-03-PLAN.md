  - under_review: amber (bg-amber-100 text-amber-800)
  - approved: green (bg-green-100 text-green-800)
  - rejected: red (bg-red-100 text-red-800)
  - onboarded: teal (bg-teal-100 text-teal-800)

**ApplicationActions** (`src/components/applications/ApplicationActions.tsx`):
- `'use client'` component
- Props: `applicationId: string, currentStatus: ApplicationStatus, currentNotes: string | null`
- Uses useTransition for pending states
- Renders status transition buttons (per CONTEXT.md: explicit buttons, not dropdown):
  - If status === 'submitted': "Move to Review" button
  - If status === 'under_review': "Approve" and "Reject" buttons
  - If status === 'approved': "Create Owner Account" button
  - If status === 'onboarded': "Onboarded" text (no actions)
  - If status === 'rejected': "Rejected" text (no actions)
- Admin notes textarea below buttons, pre-filled with currentNotes. Notes are passed along with status transition calls via the adminNotes parameter of updateApplicationStatus.
- For notes-only save: add a `saveApplicationNotes(applicationId: string, notes: string)` Server Action to applications.ts that just updates admin_notes + reviewed_by + reviewed_at without changing status. Include a "Save Notes" button.
- Create Owner Account flow (when approved): calls createOwnerFromApplication, on success displays temp password in an alert with copy-to-clipboard and warning "Save this password — it will not be shown again". Uses useRouter().refresh() after state transitions to revalidate the page.

**Application List Page** (`src/app/(owner)/dashboard/applications/page.tsx`):
- Server Component (protected by dashboard layout's requireOwner)
- Accept `searchParams` with optional `status` filter
- Query `partner_applications` ordered by created_at desc. If status filter provided, add `.eq('status', status)`.
- Render shadcn/ui Tabs at top with TabsTrigger for: All, Submitted, Under Review, Approved, Rejected, Onboarded. Each tab links to `?status=xxx` (use Link, not client-side state, to keep it a Server Component).
- Below tabs, render application cards as a vertical list:
  - Each card shows: applicant_name, applicant_email, property name (from property_basics JSONB -> propertyName), ApplicationStatusBadge, created_at formatted as date
  - Card is a Link to `/dashboard/applications/[id]`
- Empty state: "No applications found" message
- Page heading: "Partner Applications"

  - Group Hosting: experience, max group size, unique qualities
  - Logistics: check-in/out times, min stay, photo links, contact info
- Status section at top with ApplicationStatusBadge and timestamps (created, reviewed)
- Embed `<ApplicationActions applicationId={id} currentStatus={status} currentNotes={admin_notes} />` for interactive status management
- Back link: "Back to applications" link at top pointing to /dashboard/applications
