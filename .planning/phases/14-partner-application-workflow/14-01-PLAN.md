Export all 6 schemas and inferred types (e.g., `export type PropertyBasicsData = z.infer<typeof PropertyBasicsSchema>`).

**Server Actions** (`src/lib/actions/applications.ts`):

1. `submitApplication(data: unknown): Promise<{ error?: string }>` — NO auth required. Validates with ApplicationSchema.safeParse(). Uses `createClient()` (anon client, RLS allows public INSERT). Extracts contactName/contactEmail/contactPhone from logistics step for top-level applicant columns. Inserts with status='submitted'. Calls `revalidatePath('/dashboard/applications')`.

2. `updateApplicationStatus(applicationId: string, newStatus: string, adminNotes?: string): Promise<{ error?: string }>` — Requires `requireOwner()`. Fetches current status, validates transition against VALID_TRANSITIONS map:
   - submitted -> [under_review]
   - under_review -> [approved, rejected]
   - approved -> [onboarded]
   - rejected -> [] (terminal)
   - onboarded -> [] (terminal)
   Returns error for invalid transitions. Updates status, admin_notes, reviewed_by (user.id), reviewed_at, updated_at. Calls `revalidatePath('/dashboard/applications')`.

3. `createOwnerFromApplication(applicationId: string): Promise<{ tempPassword?: string; error?: string }>` — Requires `requireOwner()`. Fetches application, validates status === 'approved'. Generates temp password with `crypto.randomUUID().replace(/-/g, '').slice(0, 16) + 'A1!'`. Uses `createAdminClient().auth.admin.createUser()` with email_confirm: true and user_metadata: { role: 'owner', display_name: applicant_name }. Updates application status to 'onboarded'. Returns tempPassword. Does NOT query profiles table after creation (handle_new_user trigger is async — see research pitfall 4).

Import `createAdminClient` from `@/lib/supabase/admin`. Import `requireOwner` from `@/lib/dal`. Import `createClient` from `@/lib/supabase/server`. Use `'use server'` directive.
