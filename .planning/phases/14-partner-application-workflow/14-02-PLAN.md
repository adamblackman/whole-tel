**ApplicationForm** (`src/components/applications/ApplicationForm.tsx`):
- `'use client'` component
- State: `step` (0-indexed), `data` (Partial<StepData> accumulating all sections), `submitted` (boolean), `isPending` (useTransition), `stepError` (string | null)
- Step labels: ["Property Basics", "Capacity", "Common Areas", "Group Hosting", "Logistics"]
- Each step renders as a section within the same component (not separate files — keeps it simple):
  - Step 0 (Property Basics): propertyName (Input), location (Input), propertyType (select with villa/estate/boutique_hotel options), description (Textarea)
  - Step 1 (Capacity): maxGuests (Input number), bedrooms (Input number), bathrooms (Input number), bedConfig sub-section with king/queen/double/twin/bunk (Input number each)
  - Step 2 (Common Areas): hasPool (checkbox), hasHotTub (checkbox), outdoorSpaces (Textarea, optional), kitchenType (select: full/commercial/basic/none), notableAmenities (Textarea, optional)
  - Step 3 (Group Hosting): hasGroupExperience (checkbox), groupExperienceDetails (Textarea, conditional on hasGroupExperience), maxGroupSize (Input number, optional), uniqueForGroups (Textarea)
  - Step 4 (Logistics): checkInTime (Input), checkOutTime (Input), minimumStay (Input number), photoLinks (Textarea, optional, placeholder: "Paste Google Drive or Dropbox link to property photos"), contactName (Input), contactEmail (Input email), contactPhone (Input tel)

- **Navigation:** "Back" button (disabled on step 0) calls `setStep(step - 1)`. "Next" / "Submit Application" button validates current step with the corresponding Zod schema (client-side safeParse). If invalid, set stepError with first issue message. If valid, store step data and advance (or submit on final step).
- **Back navigation preserves data:** Each step renders with `initialData` from parent state. When going back, the step's local state initializes from the stored parent data.
- **Final submission:** On step 4 "Submit Application", call `submitApplication(fullData)` via startTransition. On success, set `submitted = true`.
- **Confirmation view:** When `submitted === true`, render a confirmation card with checkmark icon, "Application Received" heading, message that the team will review, and a link back to home page.
- **Styling:** Professional, welcoming feel. Use shadcn/ui Card for each step section, Button for navigation, Input/Label/Textarea for fields. Teal accent color consistent with brand.
