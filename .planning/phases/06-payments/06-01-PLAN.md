  <action>
1. Create `src/lib/supabase/admin.ts`:
   - Import `createClient` from `@supabase/supabase-js`
   - Export `createAdminClient()` that creates a Supabase client with `process.env.NEXT_PUBLIC_SUPABASE_URL!` and `process.env.SUPABASE_SERVICE_ROLE_KEY!`
   - Add file-level comment: "// NEVER import this in client components or Server Components. ONLY used in webhook route handlers."
   - Do NOT prefix SUPABASE_SERVICE_ROLE_KEY with NEXT_PUBLIC_ (security non-negotiable from CLAUDE.md)

2. Create `src/app/api/webhooks/stripe/route.ts`:
   - Export a single `POST` function (no GET, no other methods)
   - Read raw body with `await request.text()` FIRST (before any JSON parsing — signature verification requires raw body)
   - Get `stripe-signature` header from request
   - Call `stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!)` inside try/catch — return 400 on invalid signature
   - Handle two event types:
     - `checkout.session.completed` — fires for card (instant) and ACH (payment_status may be 'processing')
     - `checkout.session.async_payment_succeeded` — fires for ACH when funds actually clear (4 business days later)
   - Create `fulfillCheckout(session)` helper:
     - Extract `booking_id` from `session.metadata`
     - Guard: if `session.payment_status === 'unpaid'`, return early (ACH still processing)
     - Call `createAdminClient()` to get service role Supabase client
     - Update bookings table: set `status = 'confirmed'`, `stripe_session_id = session.id`, `stripe_payment_intent_id = session.payment_intent as string`, `updated_at = new Date().toISOString()`
     - Idempotency: chain `.eq('id', bookingId).eq('status', 'pending')` — only updates if still pending (duplicate webhooks are no-ops)
     - Select `id, guest_id` from the update for future email use (Plan 02)
   - Return `NextResponse.json({ received: true })` with 200 status — always return 200 to Stripe after signature verification passes (even if fulfillment has issues, to prevent retries)

  Key patterns from CLAUDE.md:
  - "Webhook (checkout.session.completed) is the authoritative booking signal, not success URL redirects"
  - "Preserve raw body for signature verification"
  - Never expose service_role key to client
