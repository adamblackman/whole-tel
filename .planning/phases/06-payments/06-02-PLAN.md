1. Add a `sendBookingConfirmationEmail(bookingId: string, guestId: string)` async function:
   - Use `createAdminClient()` to fetch full booking details:
     ```
     .from('bookings')
     .select('id, check_in, check_out, guest_count, total, properties(name)')
     .eq('id', bookingId)
     .single()
     ```
   - Fetch guest email separately from auth.users (admin client can access):
     ```
     const { data: { user } } = await supabase.auth.admin.getUserById(guestId)
     ```
   - If booking or user not found, return early (log warning)
   - Call `resend.emails.send()` with:
     - `from`: `'Whole-Tel <bookings@whole-tel.com>'`
     - `to`: user's email
     - `subject`: `Booking Confirmed - ${booking.properties.name}`
     - `react`: `BookingConfirmedEmail({ ... })` with booking details
     - Format total as currency string (e.g., "1,234.56")

2. Wrap the entire `sendBookingConfirmationEmail` call in try/catch in the fulfillCheckout function:
   - On error: `console.error('Failed to send confirmation email:', err)` — log but do NOT throw
   - Email failure must NOT prevent the 200 response to Stripe (anti-pattern from research: "Email sending should not block the webhook response")

3. Call `sendBookingConfirmationEmail(data.id, data.guest_id)` inside the `if (data)` block of fulfillCheckout (after the successful status update).
