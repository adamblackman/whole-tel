    1. **Create SplitPaymentEditor.tsx** as a 'use client' component:

       **Props:**
       ```typescript
       interface SplitPaymentEditorProps {
         bookingId: string
         total: number
         attendees: {
           invitationId: string
           name: string
           email: string
         }[]
         existingSplits: {
           invitationId: string
           amount: number
           paymentStatus: 'unpaid' | 'paid'
           stripePaymentLinkUrl: string | null
         }[]
       }
       ```

       **State:**
       - `amounts`: Record of invitationId to string — string values for input fields (avoids float issues during typing)
       - `saving`: boolean
       - `generatingLink`: string or null — invitationId currently generating
       - `error`: string or null
       - `splitsSaved`: boolean — tracks whether current amounts have been saved

       **Layout (use shadcn Input, Button, Badge):**
       - Section header: "Split Payments" with DollarSign icon
       - For each attendee:
         - Row with name (or email fallback), amount Input (type="number", step="0.01", min="0"), and status area
         - If existingSplits has a matching row:
           - Pre-fill amount from existing split
           - Show "Paid" green Badge or "Unpaid" yellow Badge based on payment_status
           - If unpaid and splits are saved: show "Generate Link" Button (or "Regenerate" if URL already exists)
           - If paid: disable amount input and show "Paid" badge, no generate button
           - If link URL exists and unpaid: show copyable link text (truncated) with a Copy button
       - Below the list:
         - "Remaining: $X.XX" display — computed as total minus sum of all parsed amounts (in real-time as user types)
         - Remaining should be green when $0.00, amber when positive, red when negative
         - "Save Splits" Button — disabled when remaining != $0.00 or saving is true
         - Error message display area

       **Behavior:**
       - On amount input change: update amounts state, recompute remaining
       - "Even split" helper button: divides total evenly, last person gets remainder (to handle rounding)
       - "Save Splits" click: call saveSplits() with parsed amounts, show error or set splitsSaved=true
       - "Generate Link" click: call generatePaymentLink(), show URL on success, show error on failure
       - Copy button: navigator.clipboard.writeText(url), brief "Copied!" feedback via useState timeout

       **Styling:** Match existing booking card style. Use Tailwind classes consistent with BookingDetails sections. Separator (hr or border-t) above the split section.

         - Only show when status === 'confirmed' AND invitations has at least one with status === 'accepted'
