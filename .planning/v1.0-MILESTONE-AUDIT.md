## Critical Issues (Blockers)

### 1. Missing Next.js Middleware (HIGH)

**Affected:** AUTH-01..06, OWNER-08

No `middleware.ts` file exists. The project's CLAUDE.md security model requires three-layer auth:
1. Middleware (optimistic UX) — **MISSING**
2. DAL `verifySession()` in Server Components/Actions — present
3. Supabase RLS at database — present

**Impact:**
- Supabase token refresh not persisted between requests (sessions will break silently)
- No optimistic redirects for unauthenticated users on protected routes
- `server.ts` line 21 comment "Middleware handles token refresh persistence" references non-existent file

**Fix:** Create `src/middleware.ts` with Supabase auth token refresh and route protection matchers.

### 2. Currency Display Bug (MEDIUM)

**Affected:** BOOK-07

`/src/app/(guest)/bookings/page.tsx` line 63-64:
```typescript
const formatCurrency = (cents: number) =>
  `$${(cents / 100).toLocaleString(...)}`
```

Database stores dollar amounts (`numeric(10,2)`), not cents. A $1,500 booking displays as $15.00.

**Fix:** Remove the `/ 100` division.
